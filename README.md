# Pier

> 把你的 Mac 当作仪表盘来读。

驻留在 macOS 菜单栏的轻量资源查看工具。点开菜单栏，在「端口 / 进程 / 系统 / 开发者」四个面板间切换——看见、理解、并就地操作，全程不用打开终端。

## 安装

```bash
brew install --cask mahui/tap/pier
```

或从 [Releases](../../releases/latest) 下载 `Pier.dmg`，拖进「应用程序」。要求 macOS 14（Sonoma）及以上，Apple 芯片与 Intel 均可；已用 Developer ID 签名并经 Apple 公证。

## 理念

- **看一眼就懂，点一下就改。** 端口、进程、系统状态一屏直读；发现问题当场就能结束进程、启停服务，不必切到终端敲命令。
- **监控工具自己不该成为负担。** 原生 SwiftUI 编写，只在面板打开或菜单栏读数开启时才采样，空闲 CPU 占用约等于零——资源监控工具不该自己就是资源大户。
- **你的数据留在本机。** 端口、进程、系统指标全部在设备上读取，从不上传。联网只为检查更新（可关）与校验许可证；没有第三方分析或广告追踪。
- **买断，不订阅。** 一次买断、永久拥有，含全部面板与后续更新。

## 功能

**端口**
- TCP / UDP 切换；同端口的 IPv4/IPv6 合并为一行并标注双栈；每 3 秒自动刷新
- 反查占用进程：GUI App 显示名称与图标，命令行 / 守护进程显示进程名与可执行文件
- 对外暴露提醒：绑定到所有网卡的端口标橙色「天线」图标
- 展开任意行查看完整命令行 / 路径 / 启动时间，并自动嗅探协议（HTTP / SSH / MySQL…）
- 右键复制 `localhost:端口`、在 Finder 中显示、拷贝 PID

**进程**
- 按 CPU 或内存实时排序的 Top 进程，带 App 名称与图标
- 展开查看完整命令行、路径与父进程，可结束或强制结束

**系统**
- CPU / 内存 / 交换 / 磁盘 / 实时网速一屏速览
- 可把实时数字钉在菜单栏图标旁，连窗口都不用开

**开发者**
- Docker / Podman 容器：查看状态与端口映射，一键启停
- Homebrew 服务：查看状态，一键启动 / 停止 / 重启

**安全的进程操作**
- 结束进程带行内二次确认，右键可「强制结束」（SIGKILL）
- 由 launchd 托管的守护进程标记 ↻，并提示「结束后会自动重启」
- root / 系统进程标记齿轮图标，结束前给红色防呆警告

**首选项**
- 开机自启、显示模式（跟随系统 / 浅色 / 深色）、自动刷新间隔、启动默认 Tab
- 菜单栏显示实时 CPU / 内存；语言切换（中文 / English / 跟随系统，即时生效）

## 为什么直接分发，不上架 App Store？

Pier 需要列出端口、读取其它进程并能结束它们——App Store 的沙箱禁止这些。所以它以 Developer ID 签名 + Apple 公证的方式直接分发，让 Gatekeeper 信任它。
